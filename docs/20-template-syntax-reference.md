# Template Syntax Reference

This is the **exhaustive** XenForo 2.3 template syntax reference: every output form,
operator, `<xf:...>` tag, template function, and value filter documented in the
official template-syntax reference. It complements the practical guide in
[`docs/05-templates.md`](05-templates.md) and the quick lookup in
[`cheatsheets/template-tags.md`](../cheatsheets/template-tags.md) — where those focus
on common usage, this page aims to be complete and precise about names and attributes.

> **Sources** (official XenForo documentation):
> - https://docs.xenforo.com/manual/reference/template-syntax
> - https://docs.xenforo.com/devs/template-basics

All XenForo tags are prefixed with the `xf:` namespace and are lowercase by
convention. Argument/variable names use `lowerCamelCase`.

---

## The three syntaxes

A XenForo template combines standard HTML with three additional syntaxes:

| Syntax | Purpose |
|--------|---------|
| `{$variableName}` | Outputs a variable. Output is automatically escaped. |
| `{{ expression }}` | Evaluates an expression: function calls, filters, math, ternaries, etc. |
| `<xf:tagName />` | A XenForo template tag for control structures, UI components, and more. |

```html
<h1>{$title}</h1>
<p>{{ phrase('welcome_message') }}</p>
<p>{{ date($xf.time, 'M j, Y') }}</p>
```

> For simple variable output, use `{$var}`. The double-curly-brace syntax `{{ }}` is
> for expressions: function calls, filters, math, ternaries, and so on. Mixing these
> up is a common mistake.

## Variable output and access

### Simple output

`{$variableName}` outputs a variable, automatically HTML-escaped:

```html
<h1>{$title}</h1>
<p>Posted by {$username}</p>
```

### Dotted / nested access

Use the dot operator to access an object property or array key. This chains for
nested structures:

```html
{$item.title}            <!-- property or array key 'title' -->
{$xf.visitor.username}   <!-- nested: visitor entity, then username -->
{$forum.node_id}
```

The same dotted access works inside `{{ }}` expressions:

```html
{{ array_first($items).title }}
```

### Method calls on objects

Within attribute/source expressions you can call entity methods directly:

```html
<xf:breadcrumb source="$category.getBreadcrumbs(false)" />
```

> Templates cannot query the database or call arbitrary PHP. Data flows from a
> controller's `$viewParams` array into template variables; that is the only way
> PHP data reaches a template. This keeps templates focused on presentation.

### How data arrives

A controller passes data through `$viewParams`; each key becomes a template
variable:

```php
public function actionIndex(): \XF\Mvc\Reply\View
{
    $viewParams = [
        'title' => 'Demo Page',
        'active' => true,
        'items' => [
            ['title' => 'First item', 'value' => 12.3456],
        ],
    ];

    return $this->view('Demo:Index', 'demo_index', $viewParams);
}
```

```html
<h1>{$title}</h1>
<xf:if is="$active"><p>Active.</p></xf:if>
<xf:foreach loop="$items" value="$item">
    <p>{$item.title}: {{ $item.value | number(2) }}</p>
</xf:foreach>
```

## Global variables (`$xf`)

In addition to view parameters, every template has access to the `$xf` global
object — information about the current request, visitor, and environment without
the controller passing it explicitly.

| Variable | Description |
|----------|-------------|
| `$xf.visitor` | The current user entity |
| `$xf.visitor.user_id` | Current user's ID (0 if guest) |
| `$xf.visitor.username` | Current user's username |
| `$xf.visitor.is_admin` | Whether the current user is an admin |
| `$xf.options.{optionName}` | XenForo option values |
| `$xf.time` | Current server timestamp |
| `$xf.language` | Current language object |
| `$xf.style` | Current style object |
| `$xf.debug` | Whether debug mode is enabled |
| `$xf.versionId` | XenForo version ID |

## Escaping vs raw output

XenForo templates automatically escape all output to prevent XSS. Both `{$var}`
and `{{ expression }}` escape HTML special characters before rendering, so you
generally do not need to think about escaping when outputting user data.

When you need to output trusted HTML (for example, rendered BBCode from the
controller), use the `raw` filter to bypass escaping:

```html
{$renderedHtml|raw}
```

For values inserted into HTML attributes, use `for_attr` for attribute-safe
escaping:

```html
<div title="{$description|for_attr}">...</div>
```

The output-control filters (full descriptions in
[Filters → Output control](#output-control)):

| Filter | Effect |
|--------|--------|
| `raw` | Disables HTML escaping. Trusted values only. |
| `escape` | Forces HTML escaping on a value. |
| `preescaped` | Marks a value as already escaped, preventing double-escaping. |
| `for_attr` | Escapes a value for safe use in an HTML attribute. |

> **Never** use `raw` with user-supplied content. Only apply it to values that
> have already been sanitized or were generated by trusted code. Using `raw` on
> untrusted input is an XSS vulnerability.

## Operators (usable in `is="..."` and `{{ }}`)

The `<xf:if>` `is` condition and `{{ }}` expressions support these operators:

| Operator | Description |
|----------|-------------|
| `&&` / `AND` | Logical AND |
| `\|\|` / `OR` | Logical OR |
| `!` | Logical NOT |
| `==`, `!=`, `>`, `<`, `>=`, `<=` | Comparison |
| `===`, `!==` | Strict comparison |
| `+`, `-`, `*`, `/`, `%` | Arithmetic |
| `.` | String concatenation |
| `? :` | Ternary |
| `?:` | Elvis (short ternary) |
| `??` | Null coalesce |
| `is` / `is not` | Type checking (e.g., `$value is empty`) |
| `instanceof` | Object type checking |

```html
<xf:if is="$xf.visitor.user_id AND $count > 10">...</xf:if>
<xf:set var="$rowClass" value="{{ $i % 2 == 1 ? 'odd' : 'even' }}" />
<xf:if is="$value is empty">...</xf:if>
```

## Control structures

### Conditionals — `<xf:if>` / `<xf:elseif>` / `<xf:else>`

Conditionally renders content. Once a condition is met, the remaining branches
are skipped. `<xf:elseif>` and `<xf:else>` are self-closing (`/`).

```html
<xf:if is="$xf.visitor.is_admin">
    <!-- Shown to admins -->
<xf:elseif is="$xf.visitor.is_moderator" />
    <!-- Shown to moderators (but not admins) -->
<xf:else />
    <!-- Shown to everyone else -->
</xf:if>
```

- `is` — The condition to evaluate.

### Loops — `<xf:foreach>`

Iterates over an array. Supports an `<xf:else>` fallback that renders when the
array is empty.

```html
<xf:foreach loop="$items" key="$key" value="$item" i="$i">
    <p>{$item.title} (#{$i})</p>
<xf:else />
    <p>No items found.</p>
</xf:foreach>
```

| Attribute | Description |
|-----------|-------------|
| `loop` | The array to iterate over. |
| `value` | Variable name for the current element. |
| `key` | Variable name for the current element's array key (optional). |
| `i` | Variable name for the 1-based iteration count (optional). |
| `if` | Condition that must be true for the loop body to render on each iteration (optional). |

The `i` counter is 1-based, so the first iteration has `$i = 1`.

### Setting variables — `<xf:set>`

Creates or aliases a variable within the template scope.

```html
<!-- Using the value attribute (expression) -->
<xf:set var="$visitor" value="{$xf.visitor}" />

<!-- Using block content (result is a string) -->
<xf:set var="$greeting">
    Hello, {$xf.visitor.username}!
</xf:set>
```

- `var` — The variable name. Must begin with `$`.
- `value` — The value or variable reference to assign.

> When using the block form (no `value` attribute), the content is **escaped and
> the result is a string**. The `value` attribute does not have this limitation,
> so use it when you need to preserve an object/array rather than a string.

### Trimming whitespace — `<xf:trim>`

Removes leading and trailing whitespace from its content.

```html
<xf:trim>
    <p>No surrounding whitespace</p>
</xf:trim>
```

### Comments — `<xf:comment>`

Adds a comment that will not appear in the rendered HTML source.

```html
<xf:comment>
    This content will not be rendered or visible in the page source.
</xf:comment>
```

## Includes and reuse

### Includes — `<xf:include>`

Includes another template. Includes share the parent's variable scope. Variables
can be passed or renamed using `<xf:map>` and `<xf:set>` children.

```html
<xf:include template="my_template" />

<xf:include template="my_template">
    <xf:map from="$localVar" to="$remoteVar" />
    <xf:set var="$newVar" value="someValue" />
</xf:include>
```

- `template` — The name of the template to include.
- Child `<xf:map from="..." to="..." />` — Renames a variable for the included template.
- Child `<xf:set>` — Injects a new variable.

### Macros

Macros are reusable blocks of template code with their own arguments and scope.

#### Defining a macro

```html
<xf:macro id="my_macro" arg-message="Default message" arg-type="info">
    <div class="block block--{$type}">
        <p>{$message}</p>
    </div>
</xf:macro>
```

- `id` — The macro identifier.
- `arg-{name}` — Declares an argument. Inside the macro it is available as
  `{$name}` (without the `arg-` prefix). Set the value to `!` to make it
  **required**; otherwise the supplied value is the default.

> Use `lowerCamelCase` for argument names. When using a macro across multiple
> templates, place the definition in its own template. **Do not use a macro tag
> as a variable — use `<xf:set>` instead.**

#### Calling a macro

Use the self-closing form to call a macro:

```html
<!-- Same template -->
<xf:macro id="my_macro" arg-message="Hello!" />

<!-- Different template: template_name::macro_name -->
<xf:macro id="my_macro_template::my_macro" arg-message="Hello!" />
```

- `id` — `macro_name` for the current template, or `template_name::macro_name`
  for a macro in a different template.
- `arg-{name}` — Values to pass as arguments.

## Template inheritance

XenForo templates support an inheritance system where a child template extends a
parent and overrides specific sections, while the parent defines the overall
structure.

### Extends — `<xf:extends>`

Declares that a template extends a parent template. It **must appear at the top
level** of the template and **must be empty**.

```html
<!-- Template: demo_custom_page -->
<xf:extends template="demo_base_page" />

<xf:extension id="content">
    <h2>Custom page content</h2>
</xf:extension>
```

- `template` — The parent template to extend.

When a child extends a parent:

1. The parent template renders and produces the page output.
2. The child's `<xf:extension>` blocks override matching extension points in the parent.
3. Any code in the child outside `<xf:extension>` blocks (such as `<xf:title>`)
   still executes for side effects, but its HTML output is discarded.

### Extension — `<xf:extension>`

Defines a named section that can be overridden by child templates. In a parent
template it provides default content; in a child template it replaces that content.

```html
<!-- Parent: demo_base_page -->
<div class="block">
    <xf:extension id="header">
        <h1>Default Header</h1>
    </xf:extension>
    <xf:extension id="content">
        <p>Default content.</p>
    </xf:extension>
</div>
```

- `id` — The extension name. Must be alphanumeric and underscores only.
- `value` — Short form for setting a single value instead of block content.

The `value` attribute is useful for passing data rather than HTML:

```html
<xf:extension id="action" value="{{ link('demo/edit', $item) }}" />
```

### Extension parent — `<xf:extensionparent>`

Renders the parent's version of an extension from within a child's override, so
you can extend rather than fully replace content.

```html
<xf:extension id="content">
    <xf:extensionparent />
    <p>Additional content appended after the parent's content.</p>
</xf:extension>
```

- `id` — Specific extension ID to render the parent of. If omitted, uses the
  current extension's ID.

### Extension value — `<xf:extensionvalue>`

Renders a named extension's content at a specific point in the template. Use this
when an extension is defined with `value` or `skipprint="true"` and you need to
output it elsewhere.

```html
<a href="{{ $extensionValue }}">
    <xf:extensionvalue id="action" />
</a>
```

- `id` — The extension ID to render.

### Wrap — `<xf:wrap>`

Renders the current template inside a wrapper template. The wrapper receives the
current template's output as a special `$innerContent` variable. This inverts the
extends relationship: the content template names its wrapper.

```html
<!-- Template: demo_page -->
<xf:wrap template="demo_wrapper">
    <xf:map from="$title" to="$pageTitle" />
    <xf:set var="$showSidebar" value="true" />
</xf:wrap>

<h2>{$title}</h2>
<p>Page content here.</p>
```

```html
<!-- Template: demo_wrapper -->
<div class="page-layout">
    <h1>{$pageTitle}</h1>
    <div class="page-content">
        {$innerContent|raw}
    </div>
    <xf:if is="$showSidebar"><aside>Sidebar</aside></xf:if>
</div>
```

- `template` — The wrapper template to use.
- Child `<xf:map>` tags rename variables for the wrapper.
- Child `<xf:set>` tags pass new variables to the wrapper.

> The wrapper template **must** output `{$innerContent|raw}` to render the wrapped
> content. The `raw` filter is required because the content is pre-escaped HTML.

### Macro extends

A macro can extend another macro from a different template, overriding specific
extension points while reusing the parent's structure. The `extends` attribute
uses the format `template_name::macro_name`.

```html
<!-- Template: demo_item_macros -->
<xf:macro id="item_card" extends="shared_macros::card" arg-item="!">
    <xf:extension id="image">
        <xf:extensionparent />
        <span class="badge">{$item.category}</span>
    </xf:extension>
</xf:macro>
```

## Navigation tags

### Breadcrumb — `<xf:breadcrumb>`

Modifies the page breadcrumb trail.

```html
<xf:breadcrumb href="{{ link('my_page') }}">{{ phrase('my_page_name') }}</xf:breadcrumb>

<xf:breadcrumb source="$category.getBreadcrumbs(false)" />
```

- `href` — The URL for the breadcrumb element.
- `source` — An array of breadcrumb data (each with `href` and `value` keys).
  Alternative to inline content.

> The root breadcrumb can be changed via the "Root breadcrumb" option in "Basic
> board information" options.

### Title — `<xf:title>`

Sets the page title, displayed in the `<h1>` tag and the browser tab.

```html
<xf:title>{{ phrase('my_page_title') }}</xf:title>
```

### H1 — `<xf:h1>`

Sets the page `<h1>` independently of `<xf:title>`. Use this when the visible
heading should differ from the browser tab title.

```html
<xf:h1>{{ phrase('welcome_back', {'name': $xf.visitor.username}) }}</xf:h1>
```

- `hidden` — If `true`, sets the H1 value without rendering it visibly.

### Description — `<xf:description>`

Sets the page's meta description.

```html
<xf:description>{{ $forum.description }}</xf:description>
```

- `meta` — If `true` (default), also sets the `<meta name="description">` tag.

### Head — `<xf:head>`

Adds content to the `<head>` section of the page. Each usage requires a unique
`option` name.

```html
<xf:head option="og_image">
    <meta property="og:image" content="{$imageUrl}" />
</xf:head>
```

- `option` — A unique key for this head entry. Subsequent entries with the same
  key overwrite earlier ones.

### Page — `<xf:page>`

Sets an arbitrary page parameter that can be read by the page container or other
templates via `page_param()`.

```html
<xf:page option="allowBookmarkControlMenu" value="1" />
```

- `option` — The parameter name.
- `value` — The parameter value.

### Page action — `<xf:pageaction>`

Defines the page action area, typically used for floating action buttons or
contextual controls.

```html
<xf:pageaction>
    <xf:button href="{{ link('threads/create', $forum) }}" icon="add">
        {{ phrase('post_new_thread') }}
    </xf:button>
</xf:pageaction>
```

- `if` — Optional condition. The page action only renders when this evaluates to true.

### Sidebar — `<xf:sidebar>`

Adds content to the page sidebar.

```html
<xf:sidebar>
    <div class="block">
        <h3 class="block-header">{{ phrase('info') }}</h3>
        <div class="block-body block-row">Sidebar content here.</div>
    </div>
</xf:sidebar>
```

- `mode` — How the content modifies the sidebar: `replace` (default), `prepend`, or `append`.

### Side navigation — `<xf:sidenav>`

Adds content to the page side navigation.

```html
<xf:sidenav>
    <div class="block">
        <h3 class="block-header">{{ phrase('navigation') }}</h3>
        <div class="block-body">
            <a href="{{ link('demo') }}">{{ phrase('demo') }}</a>
        </div>
    </div>
</xf:sidenav>
```

- `mode` — How the content modifies the side navigation: `replace` (default), `prepend`, or `append`.

### Page navigation — `<xf:pagenav>`

Renders pagination controls. This is the tag form of the `page_nav()` function.

```html
<xf:pagenav page="{$page}" perpage="{$perPage}" total="{$total}"
    link="forums" data="{$forum}" />
```

| Attribute | Description |
|-----------|-------------|
| `page` | Current page number. |
| `perpage` | Items per page. |
| `total` | Total number of items. |
| `link` | Route name for generating page URLs. |
| `data` | Entity or array passed to the link generator. |
| `params` | Additional query parameters for the link. |

## UI component tags

### Button — `<xf:button>`

Renders a button element with XenForo styling and an optional icon.

```html
<xf:button icon="save">{{ phrase('save') }}</xf:button>
```

- `icon` — The icon to display. Available named icons:
  - **Content & editing**: `add`, `edit`, `save`, `delete`, `undelete`, `copy`, `move`, `merge`, `convert`
  - **Navigation**: `prev`, `next`, `markRead`, `list`
  - **User actions**: `login`, `reply`, `quote`, `conversation`, `notificationsOn`, `notificationsOff`
  - **Moderation**: `approve`, `unapprove`, `stick`, `unstick`, `lock`, `unlock`, `disable`
  - **Files & media**: `upload`, `download`, `import`, `export`, `attach`
  - **Other**: `config`, `refresh`, `translate`, `sort`, `search`, `rate`, `vote`, `result`, `history`, `cancel`, `preview`, `purchase`, `payment`, `bolt`

### Widget — `<xf:widget>`

Includes a widget by key or class.

```html
<xf:widget key="widget_name" />
```

- `key` — The widget key (as defined in widget settings).
- `position` — Override the widget's render position.
- `class` — PHP class containing the widget definition (cannot be used with `key`).
- `title` — Widget title (used with `class`).

### Date — `<xf:date>`

Renders a dynamic `<time>` element that displays relative time (e.g., "5 minutes
ago") and updates automatically.

```html
<xf:date time="{$post.post_date}" />
```

- `time` — The timestamp to format (required).
- `class` — Additional CSS class for the `<time>` element.

### React — `<xf:react>`

Renders a reaction button for content entities that support reactions.

```html
<xf:react content="{$post}" link="posts/react" />
```

- `content` — The content entity with reaction support (required).
- `link` — The action URL for the reaction (required).
- `params` — Additional parameters for the link.
- `class` — CSS classes for the button.

### Reactions — `<xf:reactions>`

Renders the reactions summary (emoji icons and count) for a content entity.

```html
<xf:reactions content="{$post}" link="posts/reactions" />

<!-- Summary without a specific content entity -->
<xf:reactions summary="true" reactions="{$reactionCounts}" />
```

- `content` — The content entity with reaction data (required).
- `link` — URL for the full reactions list (required).
- `linkparams` — Additional parameters for the link.

### Reaction — `<xf:reaction>`

Renders a single reaction icon.

```html
<xf:reaction id="1" showtitle="true" />
```

- `id` — The reaction ID or a reaction array with a `reaction_id` key (required).
- `showtitle` — If `true`, displays the reaction title text.
- `small` — If `true`, renders in small size.
- `tooltip` — If `true`, shows a tooltip on hover.

### Profile banner — `<xf:profilebanner>`

Renders a user's profile banner image.

```html
<xf:profilebanner user="{$user}" size="l">
    <span class="memberProfileBanner-overlay"></span>
</xf:profilebanner>
```

- `user` — The User entity (required).
- `size` — Size code for the banner image (required).
- `canonical` — If `true`, uses the canonical URL.
- `hideempty` — If `true`, hides the banner when no image exists.
- `href` — Makes the banner a link.

### Show ignored — `<xf:showignored>`

Renders a "Show hidden content" link for ignored user content. Only visible to
logged-in users who have ignored users.

```html
<xf:showignored wrapperclass="block-row" />
```

- `wrapperclass` — CSS class for the wrapper element.
- `wrapper` — HTML tag name for the wrapper (e.g., `div`, `span`).
- `class` — CSS classes for the link element.

### Widget position — `<xf:widgetpos>`

Renders all widgets assigned to a named position.

```html
<xf:widgetpos id="forum_list_sidebar" />
```

- `id` — The widget position ID (required).
- `position` — Where to place the widgets: `sidebar` or `sidenav`. When set, the
  rendered widgets are injected into that page section.
- `context-*` — Dynamic context parameters passed to the widgets
  (e.g., `context-forum_id="{$forum.node_id}"`).

### Ad — `<xf:ad>`

Renders an advertisement at a named position.

```html
<xf:ad position="forum_list_above" />
```

- `position` — The ad position identifier (required).
- `arg-*` — Dynamic arguments passed to the ad macro
  (e.g., `arg-forum_id="{$forum.node_id}"`).

### Font Awesome icon — `<xf:fa>`

Inserts a Font Awesome icon.

```html
<xf:fa icon="fa-user" />
<xf:fa icon="fa-circle-notch fa-spin" />
<xf:fa icon="fa-star fa-2x" />
```

- `icon` — The Font Awesome class(es). Must include the `fa-` prefix. Supports
  modifiers `fa-spin`, `fa-pulse`, `fa-border`, `fa-flip-horizontal`,
  `fa-flip-vertical`, `fa-flip-both`, `fa-rotate-90`, `fa-rotate-180`,
  `fa-rotate-270`, and sizes `fa-xs` through `fa-9x`.

## User tags

### Avatar — `<xf:avatar>`

Inserts a user's avatar.

```html
<xf:avatar user="{$xf.visitor}" size="m" canonical="true" />
```

| Attribute | Description |
|-----------|-------------|
| `user` | The User entity. |
| `size` | Image size: `o` (384px), `h` (384px), `l` (192px), `m` (96px), `s` (48px). Falls back to `s` for invalid sizes. |
| `canonical` | Whether to use the full SEO-friendly URL. Only respected for custom avatars. |
| `notooltip` | Disable the hover tooltip. |
| `forcetype` | Force `gravatar` or `custom` avatar type. |
| `defaultname` | Fallback username when the `user` is invalid. |

### Username — `<xf:username>`

Displays a username with optional tooltip and styling.

```html
<xf:username user="{$xf.visitor}" rich="true" />
```

- `user` — The User entity.
- `rich` — Enable rich username display with group styling.
- `notooltip` — Disable the hover tooltip.
- `href` — Link URL. Disables the tooltip when set.
- `defaultname` — Fallback username when the `user` is invalid.

### Other user tags

| Tag | Purpose | Attributes |
|-----|---------|------------|
| `<xf:usertitle user="..." />` | Displays the user's title. | `user` |
| `<xf:userblurb user="..." />` | One-line summary of a user's profile. | `user` |
| `<xf:userbanners user="..." />` | The user's banners in a horizontal list. | `user` |
| `<xf:useractivity user="..." />` | The user's last action and when it occurred. | `user` |

## Asset tags

### CSS — `<xf:css>`

Includes a CSS/LESS template file, or renders inline CSS.

```html
<!-- Include a CSS template -->
<xf:css src="mycss_file.css" />

<!-- Inline CSS -->
<xf:css>
    .my-class { color: red; }
</xf:css>
```

- `src` — The CSS or LESS template to include.

### JavaScript — `<xf:js>`

Includes a JavaScript file, or renders inline JavaScript. JavaScript files are
served relative to the `/js` directory.

```html
<!-- Include a JS file -->
<xf:js src="myaddon/scripts/main.js" />

<!-- Inline JS -->
<xf:js>
    console.log('Loaded');
</xf:js>
```

| Attribute | Description |
|-----------|-------------|
| `src` | The JS file to include. Cannot be used with `prod` or `dev`. |
| `prod` | JS file to include only in production mode. |
| `dev` | JS file to include only in development mode. |
| `min` | If true, replaces `.js` with `.min.js`. Respected only in production mode. |
| `addon` | If true, uses the development JS URL. Respected only in development mode. |

### Core JavaScript — `<xf:corejs>`

Includes XenForo's core JavaScript bundle. Used by the page container template;
not normally needed in add-on templates. Has no attributes.

```html
<xf:corejs />
```

## HTML templates vs CSS/LESS templates

All template types (public, admin, email) use the same syntax. Any template can
also contain CSS/LESS — these "CSS templates" follow the same syntax but are
**compiled as stylesheets** rather than HTML.

- Include a CSS/LESS template into an HTML template with
  [`<xf:css src="..." />`](#css--xfcss).
- Inside CSS templates, the `property()` function is essential for reading style
  property values (see [Template utilities](#template-utilities)).

```html
<!-- Inside a LESS/CSS template -->
.myAddon-box {
    color: {{ property('contentText') }};
}
```

## Data lists

Data lists display tabular data using XenForo template tags. `<xf:datalist>` is a
container for `<xf:datarow>` elements and renders as a styled table.

```html
<xf:datalist>
    <xf:datarow rowtype="header">
        <xf:cell>Column 1</xf:cell>
        <xf:cell>Column 2</xf:cell>
    </xf:datarow>
    <xf:datarow>

        <xf:cell>Value 2</xf:cell>
    </xf:datarow>
</xf:datalist>
```

### `<xf:datarow>`

| Attribute | Description |
|-----------|-------------|
| `rowtype` | Set to `header` for a header row. |
| `icon` | Icon for the row (adds an extra cell). |
| `label` | Label for the row (adds an extra cell). |
| `hint` | Hint text after the label (requires `label`). |
| `explain` | Explanatory text below the label (requires `label`). |

### `<xf:cell>`

| Attribute | Description |
|-----------|-------------|
| `width` | Cell width (CSS unit: px, %, rem, etc.). |
| `colspan` | Number of columns to span. |
| `rowspan` | Number of rows to span. |
| `class` | CSS class for styling. |

Cell style classes: `dataList-cell--main` (primary content),
`dataList-cell--link` (clickable), `dataList-cell--alt` (alternative styling),
`dataList-cell--action` (action button), `dataList-cell--iconic` (with an icon),
`dataList-cell--min` (minimal width).

## Forms

Most form input tags come in two forms: a **standalone** version that renders
just the control (e.g., `<xf:textbox />`), and a **row** version that wraps it in
a label, hint, and explanation layout (e.g., `<xf:textboxrow />`). Standalone
versions accept the same control attributes (`name`, `value`, `placeholder`) but
not the row-level attributes (`label`, `hint`, `explain`).

```html
<!-- Standalone: just the input -->
<xf:textbox name="search" placeholder="{{ phrase('search') }}" />

<!-- Row: input wrapped in a form row with label -->
<xf:textboxrow name="title" value="{$title}" label="{{ phrase('title') }}" />
```

### Form container — `<xf:form>`

Wraps form elements and renders as an HTML `<form>`.

- `action` — The form submission URL.
- `ajax` — If `true`, enables AJAX form submission (inline validation errors and
  a success flash message without a full page reload).

When using `<xf:form>`, the CSRF token is included automatically.

### CSRF input — `<xf:csrf>`

Renders a hidden input containing the CSRF token. This is the tag form of the
`csrf_input()` function.

```html
<form action="{{ link('my-action') }}" method="post">
    <xf:csrf />
</form>
```

> Use `<xf:csrf>` only when building forms with plain `<form>` tags — `<xf:form>`
> adds it automatically.

### Hidden value — `<xf:hiddenval>`

Renders a hidden `<input>` field.

```html
<xf:hiddenval name="redirect" value="{{ link('forums') }}" />
```

- `name` — The form field name.
- `value` — The field value.

### Common row attributes

All `*row` form tags share these attributes for controlling the row layout:

| Attribute | Description |
|-----------|-------------|
| `label` | Label text for the form control. |
| `hint` | Hint text next to the label. |
| `explain` | Explanatory text below the control. |
| `error` | Error message displayed below the input, above the explain text. |
| `rowclass` | Additional CSS classes for the row wrapper. |
| `rowid` | HTML `id` for the row wrapper. |
| `rowtype` | Type modifier for the row styling. |
| `html` | Additional HTML after the control. |

### Form row — `<xf:formrow>`

Wraps any content in a styled form row with a label. Use when none of the
specialized `*row` tags fit.

```html
<xf:formrow label="{{ phrase('custom_input') }}"
    hint="{{ phrase('optional') }}"
    explain="{{ phrase('custom_input_explain') }}">
    <input type="text" name="custom_field" class="input" />
</xf:formrow>
```

### Info row — `<xf:inforow>`

Renders a descriptive row without a form control. Use it to display read-only
information within a form layout.

```html
<xf:inforow label="{{ phrase('status') }}">
    {{ phrase('active_since', {'date': date($user.register_date)}) }}
</xf:inforow>
```

### Text box row — `<xf:textboxrow>` (standalone `<xf:textbox />`)

```html
<xf:textboxrow name="title" value="{$title}" label="{{ phrase('title') }}"
    explain="{{ phrase('enter_a_title') }}" required="required" />
```

| Attribute | Description |
|-----------|-------------|
| `name` | Form field name. |
| `value` | Current value. |
| `type` | Input type: `text` (default), `email`, `url`, `search`, etc. |
| `placeholder` | Placeholder text. |
| `maxlength` | Maximum character length. |
| `readonly` | If set, the input is read-only. |
| `required` | If set, the field is required. |
| `ac` | Set to `single` to enable autocomplete. |

### Textarea row — `<xf:textarearow>` (standalone `<xf:textarea />`)

| Attribute | Description |
|-----------|-------------|
| `name` | Form field name. |
| `value` | Current value. |
| `autosize` | If set, the textarea automatically resizes to fit content. |
| `maxlength` | Maximum character length. |
| `code` | If set, enables code highlighting. |
| `readonly` | If set, the textarea is read-only. |

### Number box row — `<xf:numberboxrow>` (standalone `<xf:numberbox />`)

| Attribute | Description |
|-----------|-------------|
| `name` | Form field name. |
| `value` | Current value. |
| `min` | Minimum allowed value. |
| `max` | Maximum allowed value. |
| `step` | Step increment. |

### Password box row — `<xf:passwordboxrow>` (standalone `<xf:passwordbox />`)

| Attribute | Description |
|-----------|-------------|
| `name` | Form field name. |
| `hideshow` | If `true` (default), shows a toggle to reveal the password. |
| `checkstrength` | If `true`, displays a password strength meter. |

### Select row — `<xf:selectrow>` (standalone `<xf:select />`)

```html
<xf:selectrow name="category" value="{$categoryId}" label="{{ phrase('category') }}">
    <xf:option value="1">Category A</xf:option>
    <xf:option value="2">Category B</xf:option>
</xf:selectrow>

<!-- Dynamic options from a data source -->
<xf:selectrow name="category" value="{$categoryId}" label="{{ phrase('category') }}">
    <xf:options source="{$categories}" />
</xf:selectrow>
```

- `name` — Form field name.
- `value` — Pre-selected value (scalar for single, array for multiple).
- `multiple` — If `true`, allows multiple selections.
- `size` — Number of visible options.

The `<xf:option>` tag defines an option: `value`, `selected`, `disabled`, `hint`.
The `<xf:options>` tag generates options from a data source: `source`.

### Checkbox row — `<xf:checkboxrow>` (standalone `<xf:checkbox />`)

```html
<xf:checkboxrow label="{{ phrase('options') }}">
    <xf:option name="options[a]" value="1" selected="{$optionA}">
        {{ phrase('option_a') }}
    </xf:option>
    <xf:option name="options[b]" value="1" selected="{$optionB}">
        {{ phrase('option_b') }}
    </xf:option>
</xf:checkboxrow>
```

Each `<xf:option>` supports: `name`, `value`, `selected`, `label`, `hint`,
`readonly`, `disabled`.

### Radio row — `<xf:radiorow>` (standalone `<xf:radio />`)

Uses the same `<xf:option>` syntax as checkboxes.

```html
<xf:radiorow name="visibility" value="{$visibility}" label="{{ phrase('visibility') }}">
    <xf:option value="public">{{ phrase('public') }}</xf:option>
    <xf:option value="private">{{ phrase('private') }}</xf:option>
</xf:radiorow>
```

### Date input row — `<xf:dateinputrow>` (standalone `<xf:dateinput />`)

- `name` — Form field name.
- `value` — Current date value. Format: `Y-m-d`.

### Time input row — `<xf:timeinputrow>` (standalone `<xf:timeinput />`)

- `name` — Form field name.
- `value` — Current time value. Format: `H:i`.

### Date-time input row — `<xf:datetimeinputrow>` (standalone `<xf:datetimeinput />`)

- `name` — Form field name.
- `value` — Current date-time value.

### Upload row — `<xf:uploadrow>` (standalone `<xf:upload />`)

- `name` — Form field name.
- `accept` — File type filter for the file browser.

### Editor row — `<xf:editorrow>` (standalone `<xf:editor />`)

The rich text (WYSIWYG) editor.

- `name` — Form field name.
- `value` — Current BBCode content.
- `attachments` — Attachments array for the attachment manager.
- `previewable` — If `true` (default), shows a preview button.
- `maxlength` — Maximum character length.
- `removebuttons` — Array of toolbar buttons to hide.

### Code editor row — `<xf:codeeditorrow>` (standalone `<xf:codeeditor />`)

A code editor with syntax highlighting.

- `name` — Form field name.
- `value` — Current value.
- `mode` — Syntax highlighting language.
- `rows` — Number of visible rows (default: 8).

### Token input row — `<xf:tokeninputrow>` (standalone `<xf:tokeninput />`)

A tag/token input for entering multiple values.

- `name` — Form field name.
- `value` — Current values.
- `href` — URL for autocomplete suggestions.
- `min-length` — Minimum characters before autocomplete triggers (default: 2).
- `max-tokens` — Maximum number of tokens allowed.

### Telephone box row — `<xf:telboxrow>` (standalone `<xf:telbox />`)

An international telephone input with a dial code selector.

- `dialcodename` — Form field name for the dial code (required).
- `intlnumbername` — Form field name for the full international number (required).

### Prefix input row — `<xf:prefixinputrow>` (standalone `<xf:prefixinput />`)

A content prefix selector combined with a text input.

- `prefixes` — Available prefixes array (required).
- `type` — Content type for the prefixes (required).
- `prefix-name` — Field name for the prefix (default: `prefix_id`).
- `textbox-name` — Field name for the text input (default: `title`).
- `prefix-value` — Currently selected prefix.
- `textbox-value` — Current text input value.
- `href` — URL for dynamic prefix loading.

### Asset upload row — `<xf:assetuploadrow>` (standalone `<xf:assetupload />`)

A combined text input and file upload for managing assets (logos, favicons, etc.).

- `asset` — The asset type (required).
- `name` — Form field name.
- `value` — Current value.

### Captcha row — `<xf:captcharow>`

Renders a CAPTCHA challenge. Only displays when required by the visitor's
permissions.

- `force` — If `true`, always shows the CAPTCHA.
- `force-visible` — If `true`, forces the CAPTCHA to be visible.

### Submit row — `<xf:submitrow>`

Renders form submission buttons.

```html
<xf:submitrow submit="{{ phrase('save') }}" sticky="true" />
```

- `submit` — Text for the submit button.
- `icon` — Icon for the submit button.
- `sticky` — If `true`, the submit row sticks to the bottom of the viewport.

### Radio tabs — `<xf:radiotabs>`

A radio button group styled as tabs. Uses the same `<xf:option>` syntax as radio rows.

```html
<xf:radiotabs name="display_mode" value="{$displayMode}">
    <xf:option value="list">{{ phrase('list') }}</xf:option>
    <xf:option value="grid">{{ phrase('grid') }}</xf:option>
</xf:radiotabs>
```

### Captcha — `<xf:captcha>`

A standalone CAPTCHA challenge outside of a form row.

```html
<xf:captcha />
```

- `force` — If `true`, always shows the CAPTCHA regardless of visitor permissions.
- `force-visible` — If `true`, forces the CAPTCHA to be visible.
- `context` — A context identifier for the CAPTCHA.

### Redirect input — `<xf:redirectinput>`

A hidden input that tells XenForo where to redirect after form submission. Tag
form of the `redirect_input()` function.

```html
<xf:redirectinput url="{{ link('forums') }}" />
```

- `url` — Explicit redirect URL.
- `fallback` — Fallback URL if no dynamic redirect is available.
- `referrer` — If `true` (default), uses the HTTP referrer for the redirect.

### Mustache — `<xf:mustache>`

Outputs Mustache template syntax for client-side rendering. XenForo uses this
internally for JavaScript-rendered templates.

```html
<xf:mustache name="username">
    <a href="{{link}}">{{text}}</a>
</xf:mustache>
```

- `name` — The Mustache variable name (required). Must be a string literal.

### Full form example

```html
<xf:form action="{{ link('test/save') }}" class="block" ajax="true">
    <div class="block-container">
        <div class="block-body">
            <xf:textboxrow name="title" value="{$item.title}"
                label="{{ phrase('title') }}"
                explain="{{ phrase('enter_a_title') }}" required="required" />

            <xf:selectrow name="category_id" value="{$item.category_id}"
                label="{{ phrase('category') }}">
                <xf:options source="{$categoryOptions}" />
            </xf:selectrow>

            <xf:checkboxrow label="{{ phrase('options') }}">
                <xf:option name="is_enabled" value="1" selected="{$item.is_enabled}">
                    {{ phrase('enabled') }}
                </xf:option>
            </xf:checkboxrow>

            <xf:submitrow submit="{{ phrase('save') }}" />
        </div>
    </div>
</xf:form>
```

## Callbacks — `<xf:callback>`

Calls a PHP method from a template. The method must be named with a read-only
prefix, which enforces that callbacks cannot modify state.

```html
<xf:callback class="Vendor\Addon\ClassName" method="getStats" params="['sidebar']"></xf:callback>
```

- `class` — The fully qualified PHP class name.
- `method` — The method to call (must use a read-only prefix).
- `params` — An array of parameters to pass to the method.

Allowed method name prefixes:

| Category | Prefixes |
|----------|----------|
| Checks & existence | `are`, `can`, `does`, `exists`, `has`, `is`, `validate`, `verify` |
| Fetching & data | `count`, `data`, `display`, `fetch`, `filter`, `find`, `get`, `pluck` |
| Rendering & output | `print`, `render`, `return`, `show`, `view` |
| Other | `total` |

## Phrases

XenForo stores all user-visible strings as **phrases**, which are translatable and
editable by administrators. Reference a phrase by name rather than hardcoding text.

```html
<h1>{{ phrase('demo_page_header') }}</h1>

<!-- With parameters: phrase defined as "Hello, {username}!" -->
{{ phrase('greeting_with_name', {'username': $xf.visitor.username}) }}

<!-- Dynamic name (runtime) -->
{{ phrase_dynamic($phraseName) }}
```

> The `phrase()` function requires a **string literal** for the name. Using a
> variable (e.g., `phrase($myVar)`) will cause a **compile error**. Use
> `phrase_dynamic()` for variable phrase names. The literal requirement lets
> XenForo track which phrases are in use and by which templates.

## Template functions

Template functions are called within expression syntax `{{ }}`. XenForo registers
over 100 functions; the documented set follows, grouped by purpose.

### URL and linking

| Function | Description |
|----------|-------------|
| `link($route, $data, $params, $hash)` | Generates a URL from a route name. `$data` is an entity or array, `$params` is an array of query parameters, `$hash` is an optional URL fragment. |
| `link_type($type, $route, $data, $params)` | Generates a URL using a specific router type (e.g., `public`, `admin`). |

```html

<a href="{{ link('forums', $forum, {'page': 2}) }}">Page 2</a>
<a href="{{ link('posts', $post, {}, 'post-{$post.post_id}') }}">Jump to post</a>
```

| Function | Description |
|----------|-------------|

### Date and time

| Function | Description |
|----------|-------------|
| `date($timestamp, $format)` | Formats a timestamp as a date. `$format` is an optional PHP date format string. |

## Value filters

Filters transform a value using the pipe syntax: `{{ $value | filterName }}` or
`{{ $value | filterName(arg1, arg2) }}`. Filters can be chained
(`{{ $items | pluck('title') | join(', ') }}`).

### Output control

| Filter | Description |
|--------|-------------|
| `raw` | Disables HTML escaping. **Use with caution.** Only apply to values you trust. |
| `escape` | Forces HTML escaping on a value. |
| `preescaped` | Marks a value as already escaped, preventing double-escaping. |
| `for_attr` | Escapes a value for safe use in an HTML attribute. |

> The `raw` filter disables auto-escaping. Never use it with user-supplied
> content — this creates an XSS vulnerability.

### Number formatting

| Filter | Description |
|--------|-------------|
| `number($precision)` | Formats a number with locale-appropriate separators. `$precision` is decimal places (default: 0). |
| `number_short($precision)` | Abbreviated number (e.g., 1.2K, 3.4M). |
| `currency($code, $format, $showZero)` | Formats as currency. `$code` is the currency code (e.g., `USD`). |
| `file_size` | Formats bytes as a readable file size. |
| `zerofill($length)` | Pads an integer with leading zeros to `$length` digits (default: 3). |

```html
{{ $price | number(2) }}
{{ $price | currency('USD') }}
{{ $downloadSize | file_size }}
```

### String manipulation

| Filter | Description |
|--------|-------------|
| `to_lower` | Converts to lowercase. Pass `lcfirst` to lowercase only the first character. |
| `to_upper` | Converts to uppercase. Pass `ucfirst` or `ucwords` for other modes. |
| `substr($start, $length)` | Extracts a substring. |
| `replace($from, $to)` | Replaces occurrences of `$from` with `$to`. |
| `strip_tags($allowed)` | Removes HTML tags. `$allowed` is an optional string of tags to keep. |
| `nl2br` | Converts newlines to `<br>` tags. |
| `nl2nl` | Converts literal `\n` escape sequences to actual newline characters. |
| `de_camel($glue)` | Converts `camelCase` to separated words. `$glue` defaults to a space. |
| `split($delimiter, $limit)` | Splits a string into an array. `$delimiter` defaults to `,`. |
| `censor($char)` | Applies word censoring. |
| `hex` | Converts to hexadecimal representation. |
| `pad($char, $length, $postPad)` | Pads a string to `$length` with `$char`. Set `$postPad` to `true` to pad on the right. |
| `format(...$args)` | Applies `sprintf`-style formatting. |

```html
{{ $username | to_lower }}
{{ $title | substr(0, 50) }}
{{ 'camelCaseString' | de_camel }}  <!-- "camel Case String" -->
```

### Array manipulation

| Filter | Description |
|--------|-------------|
| `join($glue)` | Joins array elements into a string. `$glue` defaults to `,`. |
| `pluck($valueField, $keyField)` | Extracts a field from each element in a collection. |
| `count` | Returns the number of elements. |
| `first` | Returns the first element. |
| `last` | Returns the last element. |
| `numeric_keys_only` | Filters an array to contain only numeric keys. |

```html
{{ $names | join(', ') }}
{{ $items | pluck('title') | join(', ') }}
{{ $items | count }}
```

### Encoding

| Filter | Description |
|--------|-------------|
| `json($prettyPrint)` | Encodes as JSON. Set `$prettyPrint` to `true` for formatted output. |
| `urlencode` | URL-encodes a string. |
| `url($component, $fallback)` | Parses a URL and optionally extracts a component (e.g., `host`, `path`). |
| `url_display($component, $fallback)` | Like `url`, but first decodes punycode to UTF-8 for display. Returns `$fallback` on invalid URLs. |
| `htmlspecialchars` | Escapes HTML special characters. |
| `ip` | Converts a binary IP address to its string representation. |
| `host` | Resolves an IP address to its hostname. |
| `email_display` | Converts an email address to a UTF-8 display format. |

```html
<div data-config='{{ $config | json }}'></div>

```
